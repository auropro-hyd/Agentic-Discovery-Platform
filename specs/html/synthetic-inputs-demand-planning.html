Synthetic Input Set — Demand Planning & Distribution

12 artefacts · 5 input categories · 3 pre-engineered findings + 1 cross-domain pattern
Every document is designed to contribute to at least one finding. The findings are only surfaced when the platform reads multiple inputs together — no single document reveals a finding on its own.
🔴 Finding 1 — Forecast Ownership Conflict
3 systems, same SKUs, same period, no reconciliation. High-severity. Blocks analysis.
🔴 Finding 2 — S&OP Governance Gap
Documented process contradicts actual system behaviour. 34% override rate. High-severity.
🟡 Finding 3 — DC Allocation Model Unverified
Referenced in docs, no system evidence. Possible TSA dependency. Low-severity, amber.
🟢 Cross-Domain — Forecast Miss → O2C Impact
Demand underrun >15% → fulfilment drops 12–15 days later. Detected across WMS + O2C signals.

Design principle
No single document reveals a finding. Every finding requires cross-referencing at least two inputs — ideally from different categories (a process document contradicted by a system signal, an informal log corroborating a formal audit trail). This makes the platform's multi-source synthesis the demonstration, not just the document reading.
